01Introduction

Голосовые агенты — без audio-инженера

Год назад — месяцы работы и отдельный audio-engineering бэкграунд. Сегодня — рабочий агент за несколько минут.

Что подешевело

  • STT — быстрее и точнее, приемлемый WER для русского в реальном времени
  • LLM — тянет нюансы диалога, а не только Q&A
  • TTS — звучит по-человечески, без «robotic voice»
  • LiveKit — абстрагирует WebRTC и оркестрацию, разработчик пишет только agent-логику

Какие боли решает

  • Robotic feel — VAD по громкости даёт жёсткий turn-taking
  • «Cut you off» — агент отвечает на первой паузе, до конца мысли

Практическое применение

Направление Robots/Copilots для контакт-центров — прямой оверлеп с LiveKit.

Кандидаты STT: Yandex SpeechKit, Deepgram Nova-2, свой Whisper large-v3-turbo.

Кандидаты TTS: Yandex SpeechKit, VseGPT, ElevenLabs Multilingual v2.

02Foundations

Пайплайн: VAD → STT → LLM → TTS

TTFA (Time to First Audio) — от «клиент замолчал» до первого слова агента. До 500 мс — естественно, выше секунды — «тормозит».

Почему WebRTC, а не HTTP/WS

  • TCP — потеря пакета = head-of-line blocking, всё после него ждёт retransmit
  • WebRTC/UDP — пакет пропущен → PLC интерполирует ~20 мс, поток не рвётся
  • Плюс встроенные jitter buffer, ABR (REMB/TWCC), ICE/STUN/TURN, echo cancellation

Практическое применение

Мобильный контакт-центр (метро, лифт, LTE↔wifi) — худший кейс для HTTP/WS.

Регулируемый контур: self-hosted LiveKit Server + свой TURN в DMZ — данные не уходят наружу.

03Semantic Turn Detection

Не перебивать клиента на паузе

VAD слышит тишину и решает «очередь ответа». Semantic turn detector слушает смысл — закончена ли мысль грамматически.

  • Отдельная multilingual модель, идёт в поставке SDK
  • Работает поверх VAD, не заменяет его
  • Учитывает pause-паттерны языка — в русском своя пауза после «ну», «типа»

Включается одной строкой:

turn_detection = MultilingualModel()
~20
мс, цена включения

Trade-off

Без этой фичи — два бага сразу: агент перебивает, и точность STT падает (модель дообрабатывает начатую генерацию). Стоит включать всегда.

04Personality & Fallbacks

Характер агента + отказоустойчивость

Personality через instructions

  • Короткие фразы, минимум скобок — звучит естественнее
  • ≤3 предложения в ответе, иначе теряется внимание
  • Диалект/идиомы — явно (UK/US, RU/bilingual)
  • Эмоция — через stage directions в system prompt

FallbackAdapter

  • Каждая стадия (STT/LLM/TTS) — цепочка провайдеров по приоритету
  • Primary падает → auto-retry на следующего без разрыва сессии
  • Тест: подсунуть невалидную модель primary, смотреть цепочку в консоли

Дырка в документации

Timeouts и retry counts для FallbackAdapter нигде не описаны — «auto-retry in priority order» и всё. Для прод-контура с жёсткими SLA придётся смотреть код или задавать явно. TODO

Preemptive generation

LLM начинает генерировать ответ до конца реплики клиента — экономит 100–300 мс.

Запрещено для medical / legal / financial и других compliance-чувствительных сценариев — клиент может «переиграть» середину фразы.

Деплой

  • lk agent create — LiveKit Cloud, авто-Dockerfile, стрим логов
  • Для банковского контура — self-hosted LiveKit Server
06Tools & MCP

@function_tool и MCP-серверы

  • Docstring функции = description для LLM, тайп-хинты = контракт входов
  • MCP-сервер цепляется в AgentSession — SDK сам тянет список тулов при старте
  • Долгая операция — сказать филлер: «Одну секунду, ищу…»
  • Необратимое действие — disallow_interruptions() в начале tool-а
  • SLA на tool — держать под 2 секунды, иначе клиент бросит трубку

Практическое применение

Внутренние текстовые сервисы компании уже MCP-совместимы через mcp-proxy.

Voice-frontend получает к ним доступ без переписи логики.

Что важно

Триггеры эскалации прописываются явно в instructions примерами фраз: «если клиент просит человека», «если решение не сработало дважды».
